## Default to [YOUR-PREFERENCE: No Comments | Documented Comments]

**A comment earns its place only when removing it would mislead a future reader.**

- Add a comment only when the WHY is non-obvious — a hidden constraint, a workaround, a surprising invariant.
- Never restate what well-named code already says.
- Repo conventions (public API docs, regulated code) override the default — write the minimum that still carries the information.
